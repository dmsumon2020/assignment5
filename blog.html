Important FAQs

What is the Document Object Model (DOM)?

The Document Object Model (DOM) is like a tree that represents a web page. Each branch and leaf of the tree represents a part of the page. For example, a branch might be a <div> element, and a leaf might be the text inside that <div>. DOM turns the webpage's HTML structure into something like a family tree, where every part of the page (like text, images, and buttons) is treated as an object that can be changed or interacted using JavaScript.

How do you select an element from the DOM?

In JavaScript, we can select elements from the Document Object Model (DOM) using various methods.

  • 1. getElementById

    Method: document.getElementById(id)


    This method selects an element by its id attribute.

  • 2. getElementsByClassName

    Method: document.getElementsByClassName(className)


    This method selects all elements with a specific class name.

  • 3. getElementsByTagName

    Method: document.getElementsByTagName(tagName)


    This method selects all elements with a specific tag name, such as <p>, <div>, or <a>, and returns an HTMLCollection.

  • 4. querySelector

    Methods: document.querySelector(selector)


    This method selects the first element that matches a CSS selector. It's very flexible and can target elements by tag, class, id, or any valid CSS selector.

  • 5. querySelectorAll

    Methods: document.querySelectorAll(selector)


    This method selects all elements that match a CSS or tag selector and returns a NodeList.

What is event delegation in the context of the DOM, and why is it useful?

Event delegation is a technique in JavaScript where you attach an event listener to a parent element instead of directly to each child element. When an event occurs on any of the child elements, it bubbles up to the parent, triggering the event listener.

How Event Delegation Works

1. Event Bubbling: When an event occurs on an element (like a click), it "bubbles" up the DOM tree, meaning it goes from the target element to its parent elements. Event delegation takes advantage of this bubbling phase.

2. Single Listener: Instead of attaching multiple listeners to each child element, you attach one listener to a parent element. This parent listens for events that occur on its child elements.
